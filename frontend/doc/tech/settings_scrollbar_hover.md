# 系统设置页滚动条 Hover 问题说明

本文记录 Web 前端“系统设置页”滚动条样式调整过程中遇到的问题、排查结果和最终采用的实现原则，便于后续在其他页面复用同类交互。

相关文件：

- `frontend/src/pages/SettingsPage.vue`
- `frontend/src/components/MemberEditorDialog.vue`

## 1. 背景

系统设置页右侧主内容区 `settings-main` 是一个纵向滚动容器。设计要求是：

1. 滚动条视觉风格与成员编辑弹窗保持一致
2. 默认态较浅
3. hover 时颜色加深
4. hover 触发范围不能是整个设置页

成员编辑弹窗里的模板区域是横向滚动，滚动条样式已经较轻：

- 透明轨道
- 细滑块
- hover 时颜色略微加深

因此系统设置页最初尝试直接复用这套思路。

## 2. 问题现象

在系统设置页上，直接使用下面这类规则时，hover 体验不稳定：

```css
.settings-main::-webkit-scrollbar-thumb:hover {
  background: ...;
}
```

实际表现是：

1. 鼠标很难稳定触发 hover
2. 看起来像“hover 没有生效”
3. 即使生效，颜色差异也很弱，不明显

## 3. 根因分析

### 3.1 纵向 overlay scrollbar 命中面积太小

系统设置页是竖向滚动条，浏览器环境下使用的是 overlay scrollbar。

这类滚动条有两个特点：

1. 视觉上较细
2. 实际可稳定命中的 thumb 区域很窄

因此 `::-webkit-scrollbar-thumb:hover` 在纵向场景里不容易稳定触发。

### 3.2 横向滚动条和纵向滚动条场景不同

成员编辑弹窗里的 `member-template-grid` 是横向滚动：

- 滚动条更容易被鼠标准确压中
- 用户的注意力也集中在底部那条横向轨道上

而系统设置页是整块内容的纵向滚动：

- 滚动条靠页面最右侧
- 命中难度高
- 用户更容易把鼠标停在正文而不是滚动条上

所以两者虽然都叫“滚动条 hover”，但交互成本并不一样。

### 3.3 直接改成“容器 hover”会导致触发范围过大

中途尝试过把逻辑改成：

```css
.settings-main:hover::-webkit-scrollbar-thumb {
  background: ...;
}
```

这能稳定触发，但会带来新的问题：

1. 鼠标只要进入整个设置页内容区，滚动条就加深
2. 触发范围明显过大
3. 用户会感觉“不是 hover 滚动条，而是 hover 页面”

这不符合交互预期。

## 4. 结论

系统设置页的竖向滚动条，不适合单纯依赖 `thumb:hover`。

原因不是 CSS 语法错了，而是：

1. overlay 纵向滚动条命中面积小
2. hover 的目标区域过窄
3. 用户很难稳定把鼠标准确停在 thumb 上

## 5. 实现原则

后续处理同类问题，建议遵守下面这套原则：

### 5.1 横向局部滚动容器

例如：

- 成员编辑弹窗模板列表

这类场景可以继续使用纯 CSS：

```css
::-webkit-scrollbar-thumb:hover
```

因为它的滚动条更容易命中。

### 5.2 纵向主内容滚动容器

例如：

- 系统设置页主内容区

不要把交互建立在“必须精确 hover 到 thumb”上。

建议优先考虑：

1. 适当放大滚动条可命中宽度
2. 保持轨道透明，避免视觉过重
3. 如果确实需要更稳定的 hover 反馈，可以用“靠近滚动条区域”来驱动状态，而不是整页 hover

也就是说，判断应尽量接近“用户正在操作滚动条”，而不是“用户正在浏览页面正文”。

## 6. 当前推荐方案

当前项目里，系统设置页滚动条建议采用以下思路：

1. 视觉样式与成员编辑弹窗一致
   - 透明轨道
   - 浅色滑块
   - hover 时加深

2. 命中策略单独优化
   - 纵向滚动条可命中宽度略大于视觉宽度
   - 必要时用“滚动条区域 hover”替代 `thumb:hover`

3. 不使用整页 `:hover`
   - 避免鼠标在正文区域时滚动条也亮起

## 7. 后续复用建议

如果未来还要给其他页面做滚动条 hover：

1. 先区分横向滚动还是纵向滚动
2. 再判断滚动条是在局部控件里，还是整个页面主容器
3. 只有在“命中足够稳定”的场景下，才直接使用 `thumb:hover`
4. 对纵向主容器，优先考虑更宽的 hit area 或区域检测

一句话总结：

滚动条 hover 的难点不是配色，而是命中策略。横向局部容器可以直接靠 CSS，纵向主容器要单独处理。

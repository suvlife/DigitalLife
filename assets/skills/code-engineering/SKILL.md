# 编码与工程实现

## 适用范围
- 按需求实现功能、调试与修复缺陷、重构与性能优化。
- 前端、后端、脚本、基础设施等工程实现工作。

## 工作流程
1. 先读懂需求与现有代码结构，确认分层/依赖约束与代码规范，再动手。
2. 小步实现：先跑通最小路径，再补边界与错误处理。
3. 命名、注释密度、风格与周边代码保持一致；不引入无关改动。
4. 编写或运行自测/编译校验，确认改动可运行、不破坏既有功能。
5. 提交前自检：接口契约、错误处理、日志、安全（输入校验、越权）。

## 产物与规范
- 交付可运行代码 + 简要变更说明（改了什么、为什么、如何验证）。
- 复杂设计先写方案，评审通过再实现。

## 依赖与降级
遵循项目语言/框架与四层架构等既有约定；不确定处先提问，不臆造接口。

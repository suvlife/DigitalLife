# 测试与质量保障

## 适用范围
- 设计测试用例、编写测试计划、复现并报告缺陷、组织回归。

## 工作流程
1. 从需求/验收标准出发，列出功能点与风险点。
2. 用等价类、边界值、异常路径、状态迁移设计用例，覆盖正常与异常。
3. 用例写清：前置条件 → 步骤 → 输入 → 预期结果。
4. 缺陷报告写清：环境、复现步骤、实际 vs 预期、严重度、附证据。
5. 变更后维护回归清单，标注受影响用例。

## 产物与规范
- 交付用例表 + 测试计划 + 缺陷清单（可用 spreadsheet-studio 落表）。

## 依赖与降级
可结合自动化框架；无环境时给出手工可执行的用例步骤。

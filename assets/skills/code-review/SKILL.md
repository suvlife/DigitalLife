# 代码审查

## 适用范围
- 对提交的代码变更做结构化评审，识别缺陷与风险。

## 审查清单
1. 正确性：逻辑、边界、空值/异常、并发与竞态。
2. 契约：接口/数据结构是否与调用方一致，向后兼容。
3. 安全：输入校验、注入、越权、机密泄露、路径穿越。
4. 性能：复杂度、N+1、无谓拷贝、资源释放。
5. 可维护性：命名、分层、重复、测试覆盖、日志。

## 工作流程
1. 先理解变更意图与影响面，再逐文件评审。
2. 每条问题给出：位置 → 问题 → 触发场景 → 建议修法 → 严重级别。
3. 区分「必须修」「建议改」「可选」，最严重项排前。

## 产物与规范
- 输出按严重度排序的问题列表；无问题时明确说明已核对的维度。

## 依赖与降级
以项目代码规范文档为准；缺少上下文时先索取相关文件，不臆断。

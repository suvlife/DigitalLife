# 产品需求与规格

## 适用范围
- 将模糊诉求转化为可开发的需求：PRD、用户故事、验收标准。

## 工作流程
1. 澄清目标用户、场景、要解决的问题与成功指标。
2. 拆解为用户故事：作为<角色>，我想<目标>，以便<价值>。
3. 每个需求写明验收标准（Given/When/Then）与边界。
4. 按价值/成本排优先级（如 RICE/MoSCoW），标注依赖与风险。
5. 与设计/研发/测试对齐口径，留出灰度与度量方案。

## 产物与规范
- 交付 PRD：背景 → 目标 → 需求列表 → 验收标准 → 优先级 → 非目标。
- 正式文档交 document-studio；界面交互交 ui-ux-design。

## 依赖与降级
需求依据先经 research-brief / market-research 支撑，避免拍脑袋。

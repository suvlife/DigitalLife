# V17: 人工干预与 Agent 停止能力 - 产品文档

## 目标

为人类操作者提供实时干预手段。当 Agent 执行过程中出现不符合预期的行为时，可随时强制中断当前任务，避免无效消耗或错误扩散。

此前 Agent 一旦进入工作状态，便无法从外部打断，只能等待它完成或失败。V17 之后，操作者可在任意时刻主动停止正在运行的 Agent，Agent 停止后回到空闲状态，可继续正常参与后续调度，整个过程对系统其他部分无影响。

---

## 功能特性

### 一、随时可停止

- **主动中断**：操作者可在 Agent 执行推理或工具调用的任意阶段触发停止，无需等待当前任务完成或失败。
- **立即生效**：停止信号发出后，Agent 当前正在执行的 turn 会尽快终止，不会继续消耗模型或执行后续工具调用。
- **不依赖失败**：停止是主动操作，和 Agent 因错误进入失败状态是两件事，操作后 Agent 不进入失败状态。

### 二、停止后回归空闲

- **状态回 IDLE**：被停止的 Agent 当前 turn 结束后，Agent 本身回到空闲（IDLE）状态。
- **不阻断后续调度**：回到空闲状态后，Agent 可正常接收新的任务并继续工作，无需额外恢复操作。
- **与失败区分**：停止不触发失败流程，Agent 不会进入 FAILED 状态，也不需要通过"恢复"操作才能继续。

### 三、操作可追溯

- **活动记录留痕**：触发停止后，活动记录中会出现"被人工停止"的事件，操作者可在 Agent 活动列表中看到记录。
- **操作透明**：谁在什么时间停止了哪个 Agent，均可通过活动记录追溯。

### 四、上下文感知中断

- **历史注入**：Agent 被停止后，系统向其对话历史中写入一条系统消息，告知本轮任务已被操作者中断。
- **下次调度不混乱**：Agent 下次被调度执行新任务时，能通过历史中的中断记录感知到上一轮未正常完成，不会基于残缺的上下文继续推进。
- **LLM 可感知**：写入历史的中断说明会进入下次推理的上下文窗口，LLM 能理解当前状态，重新从新任务出发，而非延续上一轮未完成的逻辑。

### 五、范围可控

- **仅停止当前任务**：本版本只停止 Agent 当前正在执行的 turn，不影响其他 Agent 的运行。
- **不清除待处理队列**：停止操作不会删除 Agent 队列中尚未开始的任务，仅中断当前这一轮。
- **单个操作**：本版本不提供批量停止或停止整个 Team 的能力。

## 用户价值

### 1. 可以及时止损

当 Agent 明显走偏（反复调用错误工具、陷入死循环、产生无效输出）时，操作者不再只能干等，可以立即介入，避免继续消耗。

### 2. 操作不会破坏系统状态

停止后 Agent 回到空闲，不需要额外恢复步骤，下一次任务到来时仍会正常响应，整体系统不受影响。

### 3. 操作有记录

停止操作有活动记录，事后可以回溯查看某个 Agent 被停止的时间和原因，方便问题排查。

---

## 核心概念

### 停止（Stop）

由操作者主动触发，用于中断 Agent 当前正在执行的 turn。停止后 Agent 回到 IDLE，与因执行失败而进入 FAILED 状态是不同路径。

### 当前 Turn

Agent 正在执行的这一轮任务，包括从读取消息、LLM 推理到工具调用的完整执行链。停止会中断这个链路。

## 产品边界

### V17 包含

- Web Console 中对 ACTIVE 状态 Agent 提供停止操作入口
- 停止后 Agent 当前 turn 终止，Agent 回到 IDLE 状态
- 活动记录中记录"被人工停止"事件
- TUI 前端可查看停止后的状态变化（活动记录通过 WebSocket 推送）

### V17 不包含

- 批量停止多个 Agent
- 停止整个 Team 或房间内所有 Agent
- TUI 前端的停止操作入口（仅 Web Console 提供）
- 停止后自动重试或自动恢复

---

## 验收标准

### 停止操作

- ACTIVE 状态的 Agent 在详情面板中可见停止按钮
- IDLE / FAILED 状态时停止按钮不可见或不可操作
- 点击停止后，Agent 状态在界面上及时更新为 IDLE

### 状态正确性

- 停止后 Agent 不进入 FAILED 状态
- 停止后 Agent 可正常接收新任务并恢复工作
- 同一 Team 中其他 Agent 不受影响

### 上下文感知

- 停止后 Agent 的对话历史中可见"本轮被操作者中断"的系统消息
- Agent 下次执行新任务时，推理上下文中包含该中断记录，不会继续延伸上一轮未完成的逻辑

### 活动记录

- 停止操作在活动记录中留有明确记录
- 记录内容可区分"被人工停止"与"执行失败"

# V3: 多 Agent 多房间聊天 - 产品文档

## 目标

在 V2 单房间基础上扩展，支持创建多个独立聊天室，Agent 可以加入不同的聊天室，并能同时在多个聊天室中活跃，验证架构在多房间场景下的正确性。

同名 Agent（如 bob）在不同聊天室中是**相互独立的实例**：各实例的上下文仅包含所在房间的消息历史，不感知其他房间内发生的对话。这一设计保证了聊天室之间的完全隔离。

---

## 功能特性

- 各聊天室之间数据完全隔离：同名 Agent 在不同房间是独立实例，各自只感知本房间的消息历史，不感知其他房间的对话内容
- 支持配置多个独立的聊天室，每个聊天室有独立的话题和参与者
- Agent 可以加入一个或多个聊天室
- 调度器按聊天室维度并发推进对话，各房间互不干扰
- 每个 Agent 在发言前能感知当前聊天室内的其他参与者
- 消息正确路由到对应的聊天室，不会串频
- 每个聊天室独立配置最大轮次
- 完整日志记录，日志中标明房间名称

---

## 效果演示

### 配置示例

两个聊天室并发运行：

- **general** 房间：alice、bob、charlie 聊日常话题
- **tech** 房间：bob、charlie 聊技术话题

Bob 和 Charlie 同时活跃于两个房间。

### 对话示例

```
=== general 房间 ===
--- 第 1 轮 (alice @ general) ---
alice: 大家好！最近有什么有趣的事情要分享吗？

--- 第 2 轮 (bob @ general) ---
bob: 最近在研究一个分布式缓存的问题，挺有意思的。

--- 第 3 轮 (charlie @ general) ---
charlie: Bob，你有没有想过，缓存的本质是一种时间与空间的哲学博弈？

=== tech 房间 ===
--- 第 1 轮 (bob @ tech) ---
bob: charlie，你对 Python 的异步 IO 怎么看？

--- 第 2 轮 (charlie @ tech) ---
charlie: 异步 IO 让我想起了赫拉克利特——"万物皆流"，程序的执行也是如此。
```

## 使用说明

### 配置文件

编辑 `config/agents_v3.json` 设置：
- 全局 Agent 定义（名称、模型、提示词文件）
- 多个聊天室，每个聊天室配置参与者列表、初始话题和最大轮次

### 运行

```bash
python src/main.py
```

### 输出

每个房间的对话在日志中以房间名前缀区分，日志同时保存到 `logs/` 目录。

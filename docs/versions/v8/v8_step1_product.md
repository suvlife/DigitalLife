# V8: Agent 执行能力 - 产品文档

## 目标

在 V7 团队化协作的基础上，集成 Claude Code Agent SDK，赋予 Agent 真实的执行能力。Agent 不再局限于文字对话，可以在聊天室发言的过程中自主读写文件、运行代码、搜索网络，并将执行结果以自然语言反馈给其他成员，实现"说话 + 行动"的闭环。

为简化 V8 Step 1 的实现范围，系统重启后暂不自动恢复聊天室调度；重启后房间状态与历史消息可以恢复，但是否继续调度由用户显式触发。

---

## 功能特性

- **执行型 Agent**：支持将 Agent 配置为由 Claude Code Agent SDK 驱动，在发言时可按需调用文件读写、代码执行、网络搜索等能力。
- **混合聊天室**：普通对话 Agent 与执行型 Agent 可在同一聊天室共存，调度逻辑不变。
- **结果自然融入**：Agent 的执行过程对其他成员透明，只有最终整理后的结论以自然语言呈现在聊天室中。
- **能力按需配置**：每个执行型 Agent 可独立配置允许使用的工具范围，灵活控制其行为边界。

## 效果演示

### 配置示例

```json
{
  "name": "researcher",
  "use_agent_sdk": true,
  "allowed_tools": ["WebSearch", "WebFetch", "Read"],
  "system_prompt": "你是一名研究员，擅长收集和整理信息。"
}
```

### 对话示例

```
--- general ---
Alice:      听说最近 Python 3.14 发布了，有什么新特性？
researcher: 我来查一下。
            （执行中：WebSearch "Python 3.14 new features"...）
researcher: 根据官方文档，Python 3.14 主要带来了以下新特性：
            1. 模板字符串（t-strings）正式加入标准库
            2. 自由线程模式性能提升约 30%
            3. ...
Bob:        模板字符串终于来了，等了好久！
```

执行型 Agent (`researcher`) 与普通 Agent (`Alice`、`Bob`) 在同一聊天室自然协作，其他成员感知不到执行细节。

---

## 验收标准

- [ ] 支持在 Agent 配置中开启 Claude Code Agent SDK 驱动模式。
- [ ] 执行型 Agent 能在发言时调用配置范围内的工具完成真实任务。
- [ ] 执行型 Agent 与普通 Agent 可在同一聊天室正常协作，调度轮次不受影响。
- [ ] 执行过程对聊天室其他成员透明，只有最终发言可见。
- [ ] 工具范围限制生效，执行型 Agent 无法调用未配置的工具。
- [ ] 系统重启后不自动恢复聊天室调度，需由用户重新触发，以降低 Step 1 的实现复杂度。

# V1: 双 Agent 单房间聊天 - 产品文档

## 目标

实现最简单的多 Agent 聊天场景：两个 Agent 在一个聊天室中轮流对话，使用 asyncio 实现调度。

---

## 功能特性

- 固定的两个 Agent（Alice 和 Bob）
- 单个聊天室
- 轮流对话机制
- 固定轮次后自动结束
- 基于 asyncio 的调度
- 基础日志记录

---

## 效果演示

### 配置示例

**Alice 的性格设定**
- 热情开朗的朋友
- 喜欢主动开启话题
- 关心别人的生活
- 充满好奇心，喜欢提问
- 说话热情、友善，经常使用感叹号

**Bob 的性格设定**
- 安静内敛的工程师
- 回答问题比较谨慎
- 逻辑思维强，有独特的幽默感
- 说话简洁、理性，偶尔说冷笑话

### 对话示例

```
--- 第 1 轮 ---
alice: 嗨 Bob！今天天气真不错，阳光明媚，让人心情舒畅！你今天过得怎么样？

--- 第 2 轮 ---
bob: 嗨 Alice。天气确实不错，适合写代码。我正在修复一个 bug，你呢？

--- 第 3rd 轮 ---
alice: 我在整理一些文档，虽然枯燥但很有必要！你修的是什么 bug？听起来很有挑战性！

--- 第 4 轮 ---
bob: 一个并发问题，线程间的竞态条件。就像两个人同时抢最后一个甜甜圈。

--- 第 5 轮 ---
alice: 哈哈哈，这个比喻太形象了！甜甜圈确实值得抢！祝你早日解决！
```

## 验收标准

- [ ] Alice 和 Bob 能完成配置的轮次对话
- [ ] 每个 Agent 表现出配置的性格特征
- [ ] 对话记录完整保存到日志文件
- [ ] 使用 asyncio 实现调度逻辑
- [ ] 程序能正常退出

---

## 使用说明

### 配置文件

编辑 `config/agents_v1.json` 设置：
- Agent 的名称和提示词文件
- 聊天室名称和初始话题
- 最大对话轮次

### 运行

```bash
python src/main.py
```

### 输出

程序会在控制台输出每一轮的对话，最后输出完整的聊天记录。

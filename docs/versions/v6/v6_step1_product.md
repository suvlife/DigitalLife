# V6: 交互式协作 (交互简化版) 产品文档

## 1. 目标
通过引入房间分类（单聊/群聊），在不破坏现有自治模拟逻辑的前提下，实现人类与 Agent 的 1-on-1 交互能力，打通“人类下令-Agent执行”的基础链路。

## 2. 房间模态设计

### 2.1 单聊房间 (Private Room)
- **参与者**：1 Operator + 1 AI Agent。
- **交互规则**：
    - **被动触发**：Agent 不会主动开启对话，必须由操作者发起第一条消息。
    - **严格回合制**：操作者发言 -> Agent 响应 -> 结束。在 Agent 响应期间，操作者再次发送的消息将按顺序排队或由前端锁定输入。
- **应用场景**：指令下达、知识问询、性格测试。

### 2.2 群聊房间 (Group Room)
- **参与者**：多个 AI Agent。
- **交互规则**：
    - **自治模拟**：Agent 根据预设的话题和事件驱动模型自动对话。
    - **只读模式**：人类操作者在此类房间中无发言权限，仅通过 WebSocket 观察对话流。
- **应用场景**：多角色冲突演练、复杂场景模拟。

## 3. 核心功能实现

### 3.1 差异化 UI
- TUI 根据房间类型动态调整：
    - 进入 **Private Room**：底部显示输入框，提示“请输入你的指令...”。
    - 进入 **Group Room**：底部输入框禁用或隐藏，提示“当前为观察模式”。

### 3.2 通信链路
- **发送消息**：人类在 Private Room 输入内容，点击回车调用 `POST /api/rooms/{name}/messages`。
- **事件广播**：后端将消息推入 Room 历史，并向该房间的 Agent 发送特定事件。
- **响应展示**：Agent 生成的内容通过 WebSocket 推送至 TUI，渲染在左侧/右侧气泡中。

## 4. 验收标准
1. 能够区分并创建 Private 和 Group 两种类型的房间。
2. 在 Private 房间中，人类发送消息后，对应的 Agent 能够且仅有它能够给出回复。
3. 在 Group 房间中，Agent 间的对话逻辑不受人类存在的干扰（保持 V3.2 逻辑）。
4. 所有的对话数据都能通过 WebSocket 正确同步到 TUI。

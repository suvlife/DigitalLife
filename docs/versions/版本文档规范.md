# 版本文档规范

## 目录结构

```
versions/
├── 版本文档规范.md      # 本文件，记录版本管理和命名规范
├── v1/                 # V1 版本文档
├── v2/                 # V2 版本文档
├── v3/                 # V3 版本文档
└── ...
```

---

## 版本管理

每个版本对应一个独立子目录 `versions/v{n}/`，版本之间以增量方式演进，每个版本在上一版本基础上扩展功能。

| 版本 | 主题 | 状态 |
|------|------|------|
| V1 | 双 Agent 单房间 | 完成 |
| V2 | 多 Agent 单房间 | 完成 |
| V3 | 多 Agent 多房间 | 完成 |
| V3.3 | 自动化测试 | 完成 |
| V4 | Web API 与实时可视化接入 | 完成 |
| V5 | 终端可视化前端 | 完成 |

## 文档命名规范

### 格式

```
v{版本号}_step{步骤号}_{文档类型}.md
```

### 步骤说明

每个版本固定包含三类文档，按开发流程顺序编号：

| 步骤 | 文档类型 | 说明 |
|------|---------|------|
| step1 | product | 产品文档：目标、功能特性、对话示例、验收标准、使用说明 |
| step2 | technical | 技术文档：架构设计、接口定义、模块依赖、技术要点 |
| step3 | tasks | 开发任务表：任务拆分、依赖关系、复用说明、测试检查清单 |

### 示例

```
versions/v3/
├── v3_step1_product.md
├── v3_step2_technical.md
└── v3_step3_tasks.md
```

---

## 新增版本流程

1. 在 `versions/` 下创建对应子目录 `v{n}/`
2. 按顺序编写三类文档：step1 → step2 → step3
3. 在 `ROADMAP.md` 的"相关文档"章节中添加该版本的索引链接

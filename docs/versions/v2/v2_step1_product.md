# V2: 多 Agent 单房间聊天 - 产品文档

## 目标

在 V1 双 Agent 基础上扩展，支持从配置文件动态加载任意数量的 Agent（3+ 个），在单个聊天室中多个 Agent 轮流参与对话，验证架构的可扩展性。

---

## 功能特性

- 支持配置任意数量的 Agent（最少 2 个，推荐 3 个以上）
- 单个聊天室，多 Agent 轮流发言
- 每个 Agent 通过独立配置文件定义性格和行为
- Agent 在发言前能感知当前聊天室中的其他参与者
- 固定轮次后自动结束对话
- 基于 asyncio 的消息队列调度机制
- 完整日志记录

---

## 效果演示

### 配置示例

新增第三个 Agent **Charlie**，性格设定如下：
- 热爱哲学思辨的大学教授
- 喜欢引申话题到更深层的意义
- 语言优雅，引经据典
- 提问方式发人深省

### 对话示例

```
--- 第 1 轮 ---
alice: 大家好！今天天气真棒，阳光明媚！Bob 和 Charlie，你们今天打算做什么？

--- 第 2 轮 ---
bob: 嗨 Alice。修 bug。昨晚差点找到原因了，今天应该能搞定。

--- 第 3 轮 ---
charlie: 有趣。Bob，你有没有想过，修复一个 bug 与苏格拉底的"认识你自己"有几分相似——都是在追寻隐藏的真相？

--- 第 4 轮 ---
alice: 哈哈，Charlie 又开始了！不过这个比喻真的太妙了！代码里的 bug 就像人心里的盲点！

--- 第 5 轮 ---
bob: 我更愿意把它比作找漏气的轮胎。实际问题，实际解决。
```

- [ ] 支持从配置文件加载 3 个及以上 Agent

## 使用说明

### 配置文件

编辑 `config/agents_v2.json` 设置：
- 任意数量的 Agent（每个有独立的名称、提示词文件和模型）
- 聊天室名称和初始话题
- 最大对话轮次

### 运行

```bash
python src/main.py
```

### 输出

程序在控制台输出每一轮的对话内容，日志同时保存到 `logs/` 目录。

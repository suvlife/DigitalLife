# V13: 快速初始化引导 - 产品文档

## 目标

为首次使用的用户提供**自动化的配置向导**，在检测到大模型服务未配置时，自动弹出"快速初始化"引导弹窗，帮助用户快速完成必要配置并验证可用性，降低上手门槛。

本文档描述的是**前后端协作的引导能力**；核心交互发生在 Web 前端，后端提供状态检测与配置保存接口。

---

## 功能特性

### 一、启动状态检测

- **自动检测**：后端启动时自动检测 `setting.json` 中是否存在有效的 LLM 服务配置（`llm_services` 数组非空且至少有一个已启用的服务）。
- **状态标记**：若检测结果为"未配置"，系统状态中 `initialized` 字段为 `false`；否则为 `true`。
- **状态接口**：提供系统状态 HTTP API（`/system/status.json`）供前端查询，返回包含初始化状态在内的系统运行状态。

### 二、自动弹出引导弹窗

- **触发时机**：用户访问 Web Console 时，前端调用系统状态接口；若 `initialized === false`，自动弹出"快速初始化"引导弹窗。
- **无需手动跳转**：弹窗自动出现，覆盖在主界面之上，用户无需寻找配置入口。
- **弹窗位置**：居中显示，半透明遮罩背景，聚焦用户注意力。

### 三、极简配置表单

- **最小必填项**：引导弹窗仅要求用户填写三个必要字段：
    - **API 地址**（Base URL）：LLM 服务端点
    - **API Key**：鉴权密钥
    - **模型名称**：使用的模型标识
- **默认填充**：服务名称自动填充为 `"default"`；类型默认为 `"openai-compatible"`；其他参数使用系统默认值（可在后续"大模型服务管理"页面调整）。
- **智能提示**：
    - API 地址输入框提供常见服务商的端点示例（如 OpenAI、DeepSeek、Qwen）
    - API Key 输入框显示为密码类型（掩码展示）
    - 模型名称输入框提供常见模型名称提示

### 四、可用性测试

- **测试连接按钮**：表单底部提供"测试连接"按钮，用户可在保存前验证配置是否可用。
- **测试请求**：后端向目标服务发送一个最小化的推理请求（如 `"hi"`），验证 Base URL、API Key、模型名称是否正确。
- **实时反馈**：
    - **成功**：显示绿色提示，展示响应耗时
    - **失败**：显示红色提示，展示具体错误原因（如认证失败、网络不通、模型不存在）
- **独立测试**：测试请求不占用 Agent 事件队列，不写入任何消息历史或活动记录。

### 五、一键完成配置

- **完成按钮**：测试成功后，用户点击"完成"按钮，后端自动保存配置到 `setting.json`。
- **状态更新**：保存成功后，后端将 `initialized` 状态更新为 `true`，前端关闭弹窗进入正常工作界面。
- **自动设为默认**：首次配置的服务自动成为默认 LLM 服务。

### 六、跳过引导

- **跳过选项**：弹窗提供"跳过"按钮，用户可选择暂不配置，手动关闭弹窗进入主界面。
- **后续配置**：跳过后，用户可通过"系统设置 → 大模型服务管理"页面完成配置。
- **再次触发**：跳过不改变 `initialized` 状态，下次访问 Web Console 时弹窗仍会自动出现。

### 七、TUI 兼容

- **状态查询**：TUI 前端通过系统状态接口（`/system/status.json`）查询初始化状态。
- **提示信息**：未初始化时，TUI 显示提示信息："当前未配置大模型服务，请手动编辑 `~/.togo_agent/setting.json` 或通过 Web Console 完成配置"。
- **不阻塞启动**：TUI 可正常启动和展示界面，但无法执行 Agent 推理任务。

## 用户价值

### 1. 降低首次使用门槛

新用户无需了解 `setting.json` 的结构和字段含义，通过可视化引导即可完成配置。

### 2. 避免配置错误

极简表单 + 测试验证机制，确保用户填写的配置可用后才保存，避免无效配置导致的后续问题。

### 3. 快速验证可用性

"测试连接"功能让用户在保存前就能确认配置是否正确，减少反复修改的挫败感。

### 4. 平滑过渡到正常使用

配置完成后弹窗自动关闭，用户立即进入正常工作界面，体验连贯。

## 核心概念

### 初始化状态（initialized）

后端维护一个布尔状态 `initialized`，表示系统是否已完成大模型服务配置：

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| `false` | 未初始化 | `llm_services` 数组为空，或所有服务均被禁用 |
| `true` | 已初始化 | 至少有一个已启用的 LLM 服务 |

### 快速初始化弹窗

一个居中显示的模态弹窗，包含：
- 标题："快速初始化"
- 简短说明文案
- 三个必填字段输入框
- "测试连接"按钮
- "跳过"和"完成"按钮

## 产品边界

### V13 包含

- 后端启动时自动检测 LLM 配置状态
- 提供 HTTP API 查询初始化状态
- Web Console 未初始化时自动弹出引导弹窗
- 极简配置表单（三字段）
- 配置可用性测试（复用 V12 的连通性测试接口）
- 一键保存配置并更新初始化状态
- 跳过引导选项
- TUI 未初始化时的提示信息

### V13 不包含

- 多语言支持（当前仅中文）
- 分步引导流程（如选择服务商 → 选择模型 → 输入密钥）
- 配置导入功能（从其他系统导入已有配置）
- API Key 加密存储（同 V12，当前阶段明文）
- 用户账号体系与配置归属

## 后端 API 设计

### 系统状态接口

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/system/status.json` | 返回系统运行状态（含初始化状态） |

**响应示例（未初始化）**：

```json
{
  "initialized": false,
  "message": "当前未配置大模型服务"
}
```

**响应示例（已初始化）**：

```json
{
  "initialized": true,
  "default_llm_server": "qwen"
}
```

### 快速初始化配置保存

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/config/quick_init.json` | 快速初始化保存配置 |

**请求体**：

```json
{
  "base_url": "https://api.openai.com/v1",
  "api_key": "sk-xxxx",
  "model": "gpt-4o"
}
```

**成功响应**：

```json
{
  "status": "ok",
  "message": "配置保存成功",
  "detail": {
    "name": "default",
    "model": "gpt-4o"
  }
}
```

**失败响应（校验错误）**：

```json
{
  "status": "error",
  "message": "API 地址格式无效"
}
```

### 连通性测试

复用 V12 的 `/config/llm_services/test.json` 接口，使用 `mode: "temp"` 模式测试临时配置。

## 前端交互

### 弹窗触发逻辑

1. 用户打开 Web Console
2. 前端调用 `/system/status.json` 查询系统状态
3. 若响应中 `initialized === false`，立即弹出引导弹窗
4. 弹窗以模态方式显示，遮罩背景，用户无法操作主界面

### 弹窗交互流程

1. 用户填写三个必填字段
2. 点击"测试连接"按钮
3. 前端调用连通性测试接口，展示测试结果
4. 测试成功后，点击"完成"按钮
5. 前端调用快速初始化保存接口
6. 保存成功后，关闭弹窗，刷新页面进入正常状态

### 跳过引导

1. 点击"跳过"按钮
2. 弹窗关闭，用户进入主界面
3. 主界面仍可正常浏览，但 Agent 相关功能不可用（显示未配置提示）
4. 下次访问时弹窗仍会自动出现

### 状态持久化

- 弹窗已显示状态不持久化（每次访问都重新检测）
- 跳过选项不改变后端 `initialized` 状态

## 使用说明

### 首次使用快速配置

1. 启动后端服务
2. 打开 Web Console（浏览器访问后端地址）
3. 自动弹出"快速初始化"弹窗
4. 填写 API 地址、API Key、模型名称
5. 点击"测试连接"验证配置
6. 测试成功后点击"完成"
7. 弹窗关闭，系统进入可用状态

### 跳过引导后配置

1. 点击"跳过"关闭弹窗
2. 进入"系统设置 → 大模型服务管理"页面
3. 点击"新增服务"完成配置
4. 配置保存后，下次访问不再弹出引导弹窗

### 通过 TUI 使用

1. 启动 TUI 前端
2. 若未配置，TUI 显示提示信息
3. 按提示手动编辑 `setting.json` 或通过 Web Console 配置
4. 配置完成后重启 TUI 即可正常使用

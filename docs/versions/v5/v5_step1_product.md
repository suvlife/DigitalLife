# V5 产品文档 — 终端可视化前端

## 1. 背景与目标

V4 向外暴露了 HTTP 和 WebSocket 接口，但系统仍然只能通过日志观察 Agent 的发言过程，使用体验原始。V5 在不改动后端任何代码的前提下，基于 V4 的 API 开发一个**终端 TUI 前端**，让用户在命令行中实时观看多 Agent 聊天室的对话过程。

目标：打开终端，运行一条命令，即可看到聊天室列表、Agent 发言气泡实时滚动，拥有接近桌面聊天软件的观看体验。

---

## 2. 用户与使用场景

| 使用方 | 场景 |
|--------|------|
| 开发者 | 在本机终端实时观察多 Agent 聊天过程，无需打开浏览器 |
| 演示场景 | 向他人展示系统运行效果，界面直观，无需额外部署 |
| 调试排查 | 观察特定房间内的消息顺序，核查 Agent 发言是否符合预期 |

## 3. 界面布局

```
┌──────────────────────────────────────────────────────────────┐
│           Agent 聊天室观察台              12:00:05            │
├─────────────────┬────────────────────────────────────────────┤
│  聊天室          │  --- general ---                           │
│                 │                                            │
│ ● general       │  alice: 大家好！今天我们聊聊技术话题吧        │
│   [alice]       │                                            │
│   [bob]         │                    bob: 好的，我先来说说    │
│   [dave]        │                    最近 AI 领域的进展       │
│                 │                                            │
│ ○ tech          │  alice: 嗯嗯，deepseek 确实令人印象深刻！    │
│   [alice]       │                                            │
│   [bob]         │  dave: 我觉得 agent 框架更值得关注          │

│  Agent          │  状态: 运行中  消息: 12 条  已连接          │
└─────────────────┴────────────────────────────────────────────┘
```

界面分为三个区域：

- **左侧面板**：房间列表 + 当前房间成员，点击切换房间
- **右侧消息区**：实时滚动的对话气泡，自动跟随最新消息
- **底部状态栏**：连接状态、当前房间消息总数

## 4. 功能需求

### 4.1 房间列表

- 启动时通过 `GET /rooms` 拉取所有房间，展示在左侧面板
- 每个房间条目展示：房间名称、成员列表（每个成员独占一行，缩进显示）
- 当前选中的房间以高亮样式标记（`●`），其他房间用空心圆（`○`）
- 鼠标点击或键盘上下键切换选中房间，右侧随即切换显示对应房间的消息

### 4.2 消息显示

- 进入房间时通过 `GET /rooms/{room_id}/messages` 拉取历史消息，一次性渲染
- 历史消息加载完成后自动滚动到底部
- 不同 Agent 的消息按左右气泡区分：
  - **奇数序号 Agent**（如 alice、charlie）：消息靠左，绿色用户名
  - **偶数序号 Agent**（如 bob、dave）：消息靠右，青色用户名
  - **system 消息**：居中，灰色斜体，不占气泡位置
- 每条消息气泡显示内容；用户名和内容在同一气泡内

### 4.3 实时推送

- 启动后通过 `WS /ws/events` 建立 WebSocket 连接，实时接收新消息
- 收到新消息后：
  - 若当前正在查看该消息所属的房间，立即追加气泡并滚动到底部
  - 若当前查看的是其他房间，更新左侧对应房间的"新消息"角标（未读计数 +1）
- 底部状态栏实时反映连接状态（已连接 / 重连中 / 已断开）

### 4.4 Agent 列表（左侧面板底部）

- 通过 `GET /agents` 拉取 Agent 列表，在左侧面板底部展示：名称 + 模型
- 仅展示，不可交互

### 4.5 键盘操作

| 按键 | 功能 |
|------|------|
| `↑` / `↓` | 在房间列表中移动光标 |
| `Enter` | 切换到光标所在房间 |
| `PageUp` / `PageDown` | 在消息区滚动历史 |
| `q` / `Ctrl+C` | 退出程序 |

## 5. 界面风格

参考 `tui_test` 的聊天气泡实现风格：

- 整体使用 Textual 默认暗色主题（`$surface`、`$panel` 配色）
- 对方气泡：`$panel` 背景，左对齐，最大宽度 60%
- 己方气泡（偶数 Agent）：`$accent-darken-3` 背景，右对齐，最大宽度 60%
- system 消息：`$text-muted` 颜色，居中，无背景

## 6. 运行方式

### 前置条件

- V4 后端已启动（默认监听 `http://127.0.0.1:8080`）
- Python 3.11+，安装依赖：

```bash
pip install textual aiohttp
```

### 启动命令

```bash
# 从 config.json 读取后端地址（默认行为）
python tui/main.py

# 临时覆盖后端地址
python tui/main.py --base-url http://127.0.0.1:9090
```

### 退出

按 `q` 或 `Ctrl+C`。

## 7. 非功能需求

| 项目 | 要求 |
|------|------|
| 只读 | TUI 不向后端写入任何数据，纯观察模式 |
| 独立进程 | 作为独立客户端进程运行，与后端进程完全解耦 |
| 后端不可用时 | 启动时若无法连接后端，展示错误提示，不崩溃 |
| WebSocket 断开时 | 底部状态栏提示"已断开"，不崩溃；后端重连后可手动刷新 |
| 终端兼容性 | 支持标准 256 色终端，宽度建议 ≥ 100 列 |

---

## 8. 不在 V5 范围内

- 向 Agent 发送消息或干预对话（只读观察）
- 消息持久化或导出（属于独立方向）
- 多后端切换
- 用户认证

---

## 9. 验收标准

| 场景 | 预期结果 |
|------|---------|
| 启动 TUI，后端正在运行 | 左侧显示房间列表和成员，右侧显示历史消息，底部状态显示"已连接" |
| 切换房间 | 右侧消息区切换为所选房间的历史消息，自动滚动到底部 |
| Agent 在当前房间发言 | 新气泡实时追加，消息区自动滚动到最新 |
| Agent 在非当前房间发言 | 左侧对应房间出现未读数角标 |
| 切换到有未读消息的房间 | 角标清除，显示该房间的最新消息 |
| 后端未启动时运行 TUI | 显示连接失败提示，程序不崩溃 |

# V21: Agent 监督指令与控制房间 - 产品文档

## 目标

在 V20 运行中插入能力的基础上，让操作者可以直接从 Agent 的活动视图下达补充指令，而无需先找到对应的私聊房间再切换过去。

V20 解决了"私聊消息可以尽快影响 Agent 的当前工作"这个底层问题，V21 在此基础上进一步解决"如何更自然地发出这条消息"——让操作者不离开当前观察界面，就能完成一次快速干预。

---

## 功能特性

### 一、在活动视图中直接下指令

操作者在 Agent 的活动视图（状态/活动面板）中，可以直接输入补充要求、纠偏建议或临时约束，并直接发送，而不需要先切换到对应的私聊房间。

发出的指令会通过已建立的私聊房间传达给 Agent，并利用 V20 的立即注入能力，在 Agent 当前工作过程中尽快生效。

### 二、首次使用自动建控制房间

当操作者第一次向某个 Agent 下达追加指令时，如果私聊控制房间尚不存在，系统会自动为该 Agent 创建一个对应的私聊房间，并将指令发送进去。

操作者无需提前手动为每个 Agent 建立私聊房间，也不会因为默认界面预先创建过多空房间而显得混乱。

### 三、指令尽快生效

追加指令发出后，行为与 V20 中的"立即注入"消息一致：

- Agent 正在运行时，指令会在当前步骤完成后的自然衔接点尽快注入
- Agent 当前空闲时，按普通私聊消息处理，触发正常的轮次流转

### 四、正式沟通规则不变

操作者虽然能在活动视图中观察 Agent 的执行轨迹，但：

- Agent 若要正式回复操作者或团队，仍需通过房间消息明确表达
- "看得到执行过程"与"正式沟通"是两件事，不互相混淆
- 活动视图只是一个更方便的指令入口，底层沟通渠道仍是私聊房间

### 五、全过程可追溯

通过活动视图发出的所有追加指令，以及 Agent 的后续响应，都会正常保存在对应的私聊控制房间中，可随时回看。

---

## 用户价值

### 1. 干预更流畅

以前操作者要追加一条指令，需要先定位到目标 Agent 的私聊房间，切换过去，再发消息。现在可以直接在活动视图输入，动作路径缩短了一半。

### 2. 无需提前规划房间结构

以前如果想让每个 Agent 都有独立的私聊房间，需要提前手动创建。V21 改为按需自动建立，用到才出现，不用的不占界面空间。

### 3. V20 能力的自然延伸

V21 并不新增指令底层传达机制，而是为 V20 的立即注入能力提供了一个更自然的入口。操作者无需手动切换房间，整个"观察 → 发现问题 → 追加指令"的流程更加连贯。

---

## 核心概念

### 控制房间

指操作者与某个 Agent 之间的私聊房间，作为下达追加指令和 Agent 正式回复的专用通道。V21 中，控制房间可在首次使用时由系统自动创建，而不需要预先手动建立。

### 活动视图指令入口

指 Agent 活动面板中新增的轻量输入区域，允许操作者在不离开当前观察界面的情况下，直接向该 Agent 发送追加指令。本质上等同于在对应的私聊房间中发送一条立即注入消息。

### 按需建房

控制房间不在系统初始化时批量创建，而是在操作者首次通过活动视图向某个 Agent 下达指令时，由系统自动建立。

## 效果演示

### V21 之前

```text
场景：操作者发现 Agent 正在执行 execute_bash，结果方向明显偏了

步骤：
1. 操作者看到活动视图里的执行过程，发现有问题
2. 操作者需要找到左侧房间列表，定位到该 Agent 的私聊房间
3. 如果还没有私聊房间，需要先手动创建
4. 切换到房间，发送纠偏消息
5. 消息进入普通等待队列，等 Agent 整轮结束后才生效（V20 之前）
   或选择立即注入，尽快生效（V20 之后）
```

### V21 之后

```text
场景：操作者在活动视图中观察到 Agent 方向偏了

步骤：
1. 操作者在当前活动视图下方的输入框里，直接输入纠偏指令
2. 点击发送（默认标记为立即注入）
3. 如果该 Agent 还没有私聊房间，系统自动创建
4. 指令通过私聊房间以立即注入方式发出，在当前步骤完成后尽快生效
5. 操作者无需离开活动视图，继续观察后续执行
```

## 产品边界

### V21 包含

- Agent 活动视图中新增追加指令输入区域
- 首次使用时自动创建对应的私聊控制房间
- 追加指令默认以"立即注入"方式发送（复用 V20 能力）
- Agent 的正式回复仍通过房间消息进行，不变
- 指令及响应全程保存在私聊控制房间，可追溯

### V21 不包含

- 修改 Agent 的正式回复渠道或格式
- 在群聊房间中新增类似的快捷指令入口
- 活动视图内的双向对话（操作者看到 Agent 回复）：回复仍需到私聊房间查看
- Agent 主动向操作者推送"请求更多信息"的能力
- 强制中断当前 Agent 动作的能力

## 验收标准

### 活动视图指令入口

- 在 Agent 活动视图中，存在可输入文本并发送的操作区域
- 发送的消息可通过对应的私聊控制房间准确送达 Agent

### 按需建房

- 操作者第一次通过活动视图向某 Agent 发送追加指令时，若私聊房间不存在，系统自动创建
- 创建后继续使用同一房间，不重复建立
- 已有房间的情况下，直接复用，不重新创建

### 立即注入默认行为

- 通过活动视图发出的追加指令，默认以立即注入方式发送
- Agent 正在运行时，指令在当前步骤完成后的衔接点参与后续处理
- Agent 空闲时，按正常私聊消息处理

### 可追溯性

- 通过活动视图发出的所有指令，都可在对应私聊房间的消息列表中查到
- 指令的发送时间、内容、状态与普通私聊消息一致

### 正式沟通不受影响

- Agent 的正式回复仍通过房间消息体现，不因新入口改变
- 群聊房间消息流转不受影响
